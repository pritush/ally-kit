🎨 Ally Widget Configuration Guide

📋 Overview

The Ally Widget now supports feature-level configuration, allowing developers to enable or disable specific accessibility features based on their project requirements. This gives you complete control over which features are available to your users.

🚀 Quick Start

Add the configuration object before including the widget script: 

<script>
  window.AllyKitConfig = {
    features: {
      textSize: true,
      highContrast: true,
      screenReader: false,  // Disable screen reader
      adhd: false           // Disable ADHD mode
      // ... other features
    }
  };
</script>

<script src="https://cdn.jsdelivr.net/gh/pritush/ally-kit@master/ally-kit-min.js"></script>

🎛️ Available Features

All features are enabled by default. Set any feature to false to disable it:

textSize Adjust text size from 100% to 140%
highContrast High contrast modes including dark mode
textSpacing Letter and word spacing adjustments
lineHeight Line height from 1x to 2x
dyslexia OpenDyslexic font for better readability
adhd Focus mode to reduce distractions
saturation Color saturation control and grayscale
invert Invert colors for visual comfort
bigCursor Enlarged cursor for better visibility
hideImages Hide all images and background images
pauseAnimation Stop all animations and transitions
highlightLinks Highlight all clickable links
screenReader Built-in screen reader with speech synthesis

⚙️ Complete Configuration Object

Here's the full configuration object with all available options:

window.AllyKitConfig = {
  // Position and Layout
  position: "right",           // "left" or "right"
  buttonOffset: "24px",        // Distance from screen edge
  panelWidth: "560px",         // Width of accessibility panel
  
  // Feature Toggles (all default to true)
  features: {
    textSize: true,
    highContrast: true,
    textSpacing: true,
    lineHeight: true,
    dyslexia: true,
    adhd: true,
    saturation: true,
    invert: true,
    bigCursor: true,
    hideImages: true,
    pauseAnimation: true,
    highlightLinks: true,
    screenReader: true
  }
};
💡 Pro Tip: You only need to specify features you want to disable. Features not mentioned in the config will remain enabled by default.
⚠️ Important: Make sure to define window.AllyKitConfig before including the ally-kit script, otherwise your configuration won't be applied.

🔧 Partial Configuration

You can also use partial configuration - only specify what you want to change:

// This will disable only these 3 features, all others remain enabled
window.AllyKitConfig = {
  features: {
    adhd: false,
    hideImages: false,
    invert: false
  }
};

📱 Testing Your Configuration

On this page, we've disabled ADHD mode, Color Inversion, and Hide Images features. Open the accessibility widget (bottom right corner) to see how your configuration affects the available features.

📚 Use Cases

  • Corporate Sites: Disable features that conflict with branding guidelines
  • E-commerce: Keep product images visible by disabling hideImages
  • Educational: Focus on reading features (dyslexia, text spacing, line height)
  • Blogs: Enable all text and reading-focused features
  • Web Apps: Disable features that might interfere with UI (like ADHD mask)
