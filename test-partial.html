🎯 Partial Configuration Test

Partial Configuration Active: This page disables only 3 specific features. All other features (10 total) remain enabled by default.
💡 Key Concept: You only need to specify features you want to disable! This is the most developer-friendly approach.

Test Instructions

  1. Open the accessibility widget (button in bottom-right corner)
  2. Verify that you see 10 feature buttons (not 13)
  3. Confirm these 3 features are missing: ADHD Mode, Hide Images, Invert Color

Expected Results

✅ Enabled Features (Should Appear):

  1. Text Size
  2. High Contrast
  3. Text Spacing
  4. Line Height
  5. Dyslexia Friendly
  6. Saturation
  7. Big Cursor
  8. Pause Animation
  9. Highlight Links
  10. Screen Reader

❌ Disabled Features (Should NOT Appear):

  1. ADHD Mode
  2. Hide Images
  3. Invert Color

Configuration Code

Notice how simple this is - only 3 lines!

window.AllyWidgetConfig = {
  features: {
    adhd: false,
    hideImages: false,
    invert: false
  }
};

Why This Approach?

  • Clean & Simple: Only specify what you need to change
  • Maintainable: Easy to understand at a glance
  • Future-proof: New features will be enabled automatically
  • Less Code: No need to list all 13 features

Use Case Example

This configuration is perfect for a web application where:

  • ADHD Mode disabled: The reading mask might obscure important UI elements
  • Hide Images disabled: Images and icons are functional, not decorative
  • Invert Colors disabled: Your app has a carefully designed color scheme
